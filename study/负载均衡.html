负载均衡

安锐2026/4/19大约 4 分钟

负载均衡

⚙️ 2. 安装与启停

Ubuntu/Debian

# 安装
sudo apt update
sudo apt install nginx

# 启动
sudo systemctl start nginx

# 开机自启
sudo systemctl enable nginx

# 停止
sudo systemctl stop nginx

# 重启 (平滑重启，不影响正在处理的请求)
sudo systemctl reload nginx
# 或
sudo nginx -s reload

# 检查配置是否正确
sudo nginx -t

CentOS/RHEL

# 添加 EPEL 仓库 (如果需要)
sudo yum install epel-release
# 安装
sudo yum install nginx
# 启停命令与 Ubuntu 相同 (systemctl)

📁 3. 核心概念与文件结构

  • 主配置文件: /etc/nginx/nginx.conf
  • 默认服务器块 (虚拟主机) 配置: /etc/nginx/sites-available/default (Ubuntu) 或 /etc/nginx/conf.d/default.conf (CentOS)
  • 自定义配置目录: 通常将自定义配置放在 /etc/nginx/conf.d/ 或 /etc/nginx/sites-available/ (并通过软链接到 /etc/nginx/sites-enabled/)
  • 日志文件:
    • 访问日志: /var/log/nginx/access.log
    • 错误日志: /var/log/nginx/error.log

🧪 4. 实战案例

在进行任何配置修改后，务必先执行 sudo nginx -t 测试配置，然后再 sudo systemctl reload nginx 重新加载。

案例一：提供静态网站

目标：将本地 /data/www 目录下的 HTML 文件通过 Nginx 提供访问。

  1. 创建目录和测试文件：

    sudo mkdir -p /data/www
sudo echo "<h1>Hello, World! This is my static site.</h1>" | sudo tee /data/www/index.html

  2. 编辑默认配置文件 (/etc/nginx/sites-available/default 或 /etc/nginx/conf.d/default.conf)，找到 server 块，修改 location / 部分：

    server {
    listen 80; # 监听 80 端口
    server_name _; # 匹配所有域名，也可改为你的域名，如 server_name example.com;

    # 指定静态文件根目录
    root /data/www;
    index index.html index.htm;

    location / {
        # 尝试提供请求的文件，如果找不到则返回 404
        try_files $uri $uri/ =404;
    }
}

  3. 保存文件并重启 Nginx。

第七步：验证效果

在浏览器访问你的服务器 IP 或配置的域名：

http://your_server_ip

网页应显示 “Hello, World! This is my static site.”

案例二：配置反向代理

目标：将所有访问 http://your_server_ip/app/ 的请求，转发到运行在 http://localhost:3000 上的 Node.js 应用。

在 server 块中添加以下 location 块：

server {
    listen 80;
    server_name _;

    # 处理 /app/ 路径的请求
    location /app/ {
        # 设置反向代理
        proxy_pass http://localhost:3000/; # 末尾的 / 很重要，它会将 /app/ 前缀去除后再转发
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 其他请求仍然可以由静态文件处理
    location / {
        root /data/www;
        index index.html index.htm;
    }
}

验证效果

在浏览器访问：

http://your_server_ip/app/

Nginx 会将请求透明地转发给 localhost:3000 并返回响应。

案例三：配置负载均衡

目标：将流量均匀地分发到三个后端应用服务器上。

  1. 在 http 块中（通常在 nginx.conf 的顶部），使用 upstream 定义一个后端服务器组：

    http {
    # 定义一个名为 backend_servers 的 upstream 组
    upstream backend_servers {
        # 默认使用轮询 (round-robin) 策略
        server 192.168.120.201:8080 weight=1; # weight 表示权重，可用于性能不均的服务器
        server 192.168.120.202:8080 weight=1;
        server 192.168.120.203:8080 weight=2; # 该服务器接收的请求将是其他的两倍
        # backup 表示备份服务器，只有在其他服务器都宕机后才启用
        # server 192.168.120.204:8080 backup;
    }

    # ... 其他配置 ...
    include /etc/nginx/conf.d/*.conf;
}

  2. 在 server 块中，将特定请求代理到这个 upstream 组：

    server {
    listen 80;
    server_name _;

    location / {
        proxy_pass http://backend_servers; # 注意这里指向 upstream 的名字
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        # ... 其他 proxy_set_header 指令 ...
    }
}

验证效果

在浏览器反复访问：

http://your_server_ip/

Nginx 会将请求按权重轮询分发到 201, 202, 203 三台后端服务器。你需要在后端服务器的应用日志中观察请求的接收情况。

📚 附录：常用命令与配置片段

🔧 常用命令

命令说明
sudo nginx -t测试配置文件语法是否正确 (非常重要！)
sudo systemctl start nginx启动 Nginx 服务
sudo systemctl stop nginx停止 Nginx 服务
sudo systemctl restart nginx重启 Nginx 服务 (会中断连接)
sudo systemctl reload nginx平滑重载配置文件 (不中断服务)
sudo systemctl status nginx查看 Nginx 服务状态
sudo tail -f /var/log/nginx/error.log实时查看错误日志
sudo tail -f /var/log/nginx/access.log实时查看访问日志

⚙️ 常用配置片段

1. 重定向 HTTP 到 HTTPS

server {
    listen 80;
    server_name your_domain.com;
    return 301 https://$server_name$request_uri; # 301 永久重定向
}

2. 简单的 SSL 配置

server {
    listen 443 ssl http2;
    server_name your_domain.com;

    ssl_certificate /path/to/your/fullchain.pem;
    ssl_certificate_key /path/to/your/privkey.pem;

    # ... 其他配置 ...
}

3. 限制访问

location /admin/ {
    # 只允许指定网段访问
    allow 192.168.1.0/24;
    allow 10.8.0.1;
    deny all; # 拒绝所有其他 IP
    # ... 其他配置 ...
}

4. 客户端 Body 大小限制 (解决 413 错误)

server {
    # ...
    client_max_body_size 100M; # 允许上传最大 100M 的文件
    # ...
}

🚀 下一步

  1. 深入学习配置语法：理解 server, location, upstream 等块的匹配规则和优先级。
  2. 性能调优：学习调整 worker_processes, worker_connections, gzip 等参数。
  3. 安全性：配置更严格的 SSL 协议、CSP 头、隐藏 Nginx 版本号等。
  4. 高可用：结合 Keepalived 实现 Nginx 自身的高可用。
